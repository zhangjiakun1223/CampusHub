## 一、本阶段目标

2. 编写规范的需求规格说明书
3. 完成用例建模与用户故事编写

## 二、任务说明

### 任务 1：用户调研（4 小时）

#### 1.1 设计调研方案
- 确定调研目标：了解校园互助的真实需求场景
- 设计调研问卷（10-15 个问题，课程统一设计）
- 准备访谈提纲（5-8 个开放式问题）

#### 1.2 执行调研
- **问卷调研：** 收集至少 20 份有效问卷
- **用户访谈：** 每位团队成员至少访谈 1 位同学（团队共 ≥ 4 人次）
- 记录原始访谈笔记

#### 1.3 AI 对比实验 —— "AI 需求盲区挑战"

这是本阶段的**关键实验**，请严格按以下步骤执行：

**第一步：AI 生成**
- 向 AI 提供以下 Prompt（或团队优化后的版本）：

> "请为一个校园互助服务平台生成完整的需求规格说明书，平台支持学生发布和接单各类互助需求（快递代取、学习辅导、二手交易、组队匹配等），目标用户为大学生。请按 IEEE 830 标准格式输出。"

- 保存 AI 生成的完整输出

**第二步：人工调研**
- 基于问卷和访谈结果，团队独立整理需求（不参考 AI 输出）

**第三步：对比分析**
- 将 AI 生成的需求与人工调研结果对比，填写下表：

| 对比维度 | AI 生成的需求 | 人工调研发现的需求 | 分析 |
|---------|-------------|----------------|------|
| AI 正确覆盖的需求 | | | |
| AI 遗漏的需求 | | | 为什么 AI 会遗漏？ |
| AI 捏造的需求 | | | 这些需求真的存在吗？ |
| AI 表述有歧义的需求 | | | 歧义在哪里？ |

### 任务 2：需求规格说明书（4 小时）

基于调研结果和 AI 辅助，编写正式的需求规格说明书。

**文档结构要求（参考 IEEE 830）：**

1. 引言（目的、范围、定义、参考文献）
2. 总体描述（产品视角、用户特征、约束、假设）
3. 功能需求（按模块组织，每条需求含编号、描述、优先级）
4. 非功能需求（性能、安全、可用性、可维护性）
5. 需求追踪矩阵（需求 → 用例的对应关系）

**标注规则：**
- 每条需求必须标注来源：`[AI生成]`、`[人工补充]`、`[AI生成+人工修改]`
- AI 生成后经人工修改的需求，需简要说明修改了什么

### 任务 3：用例建模与用户故事（3 小时）

#### 3.1 用例图
- 绘制系统级用例图（至少覆盖 3 个角色、15 个用例）
- 为 3 个核心用例编写详细用例描述（含前置条件、基本流、替代流、后置条件）

#### 3.2 用户故事
- 编写至少 15 条用户故事，格式：

```
作为 [角色]，我希望 [功能]，以便 [价值]。
验收标准：
1. ……
2. ……
```

- 对每条用户故事进行优先级排序（MoSCoW 方法：Must / Should / Could / Won't）

#### 3.3 AI 辅助实验
- 让 AI 生成用例和用户故事，与人工版本对比
- 重点关注，并**举例回答**：AI 生成的内容是否遗漏了异常流？验收标准是否具体可测试？

### 任务 4：需求评审（1 小时）

- 团队内部交叉评审需求文档
- 检查需求之间是否存在冲突或矛盾
- 验证需求的可行性（在 10 周内是否能实现）
