# 《AI 协作契约》

团队名称：四个火枪手

⽇期：2026.4.16

## 第⼀条：AI 使⽤原则
- AI 是⼯具，⼯程师是责任主体
- 所有 AI 输出必须经过⼈⼯审查后才能合⼊项⽬

## 第⼆条：AI 使⽤范围
- 允许使⽤ AI 的场景：
  - 生成代码模板（如CRUD、表单页面）
  - 辅助生成文档初稿（需求、设计、测试报告等）
  - 生成单元测试用例
  - 解释不熟悉的代码或技术概念
  - 辅助调试（分析错误日志）
- 禁⽌直接使⽤ AI 输出（需额外审查）的场景：
  - 核心业务逻辑
  - 数据库表结构设计
  - 安全相关代码（加密，防sql注入）
  - 架构决策（技术选型、模块划分）
  - 反思日志
- 完全禁⽌使⽤ AI 的场景：
  - 代替团队进行工程判断

## 第三条：过程记录要求
- 所有与 AI 的交互 Prompt 必须保留⽇志
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]
- 每阶段填写《AI 协作反思⽇志》

## 第四条：代码合⼊规则
- AI ⽣成的代码必须通过⾄少⼀名团队成员的 Code Review
- 每位成员每个编码阶段⾄少有 2 个完全⼿写的核⼼函数

## 第五条：违约处理
- 在团队例会上公开说明原因并检讨
- 修复问题并整理团队文档
