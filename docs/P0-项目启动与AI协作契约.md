### 任务 2：AI 工具选型

团队讨论并选择本项目使用的 AI 工具链。需要考虑：

**需要选择的工具类型：**
1. **AI 编程助手**（如 GitHub Copilot、Cursor、Claude Code、通义灵码等）
2. **AI 对话工具**（如 ChatGPT、Claude、文心一言等，用于文档生成、需求分析）
3. **AI 代码审查工具**（如 CodeRabbit、AI Code Review 等）
4. **AI 测试工具**（如 AI 测试用例生成工具）

**选型要求：**

- 每种类型至少评估 2 个候选工具
- 记录选择理由（功能、易用性、成本、团队熟悉度）或不选择某工具的理由
AI编程助手：GitHub Copilot 通义灵码 
理由：GitHub Copilot：
  功能强大，能实时生成代码片段，辅助后端 Java、前端 Vue/HTML 等开发。
  与 IDEA深度集成，开箱即用，适合快速搭建业务逻辑。
  团队整体熟悉度高，是目前企业级项目最常用的 AI 编码工具。
  通义灵码：
  对中文语境支持更好，适合快速生成中文注释、文档、简单工具类。
  在代码补全、语法检查方面表现优秀，与 Copilot 形成互补。
  免费额度足够，成本低，适合课程项目使用。
AI对话工具：ChatGPT Deepseek
理由：ChatGPT：
  能力全面，适合需求分析、用例建模、文档生成、架构方案讨论。
  输出逻辑清晰，可直接生成结构化的 P0/P1/P2 文档内容。
  团队熟悉度高，沟通成本低。
  DeepSeek：
  针对中文内容优化，适合撰写中文报告、需求规格说明书。
  推理能力强，可用于复杂业务逻辑的需求拆解。
  免费且稳定，适合大作业长期使用。
AI代码审查工具：Codereview.cn CodeRabbit
理由：Codereview.cn：
  功能强大，支持代码审查、测试用例生成、性能测试等。
  团队熟悉度高，沟通成本低。
  CodeRabbit：
  针对 Java 项目优化，适合代码审查、测试用例生成。
  免费且稳定，适合大作业长期使用。
AI测试工具：TestGPT + 通义测试助手
理由：TestGPT：
  基于输入的代码或需求，自动生成单元测试、接口测试用例。
  适合快速生成 JUnit 测试、API 测试脚本，提高测试效率。
  与 GitHub 集成方便，可嵌入开发流程。
  通义测试助手：
  中文语境支持好，能根据需求文档生成测试场景、测试步骤。
  适合从需求角度验证功能，辅助测试负责人设计测试用例。
  免费稳定，适合课程项目。

### 任务 3：签署《AI 协作契约》

团队共同制定并签署一份《AI 协作契约》，明确以下内容：

```
《AI 协作契约》模板

团队名称：__________
日期：___2026年4月15日_______

第一条：AI 使用原则
- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

第二条：AI 使用范围
- 允许使用 AI 的场景：___代码、文档、测试用例_______
- 禁止直接使用 AI 输出（需额外审查）的场景：__代码________
- 完全禁止使用 AI 的场景：____调查结果，产品调研结果______

第三条：过程记录要求
- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]
- 每阶段填写《AI 协作反思日志》

第四条：代码合入规则
- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数

第五条：违约处理
- 团队内部约定的违约处理方式

全体成员签名：张加坤 刘明轩 朱鸣涛 段朝睿
```

**思考引导：** 为什么需要这份契约？如果没有审查规则，可能会出现什么问题？请回忆课堂上讲到的"工具依赖而能力退化"现象。

### 任务 4：搭建项目基础设施

1. **创建 Git 仓库**
   - 初始化项目仓库（GitHub/GitLab/Gitee 均可）
   - 配置 `.gitignore`（根据选择的技术栈）
   - 编写 `README.md`（项目简介、团队成员、技术栈）
   - 确定分支策略（建议：main + dev + feature 分支）

2. **搭建基础 CI 流水线**
   - 配置自动化构建（可使用 GitHub Actions / GitLab CI）
   - 至少包含：代码编译检查 + 代码风格检查

3. **初始化项目结构**
   - 创建基础目录结构
   - 配置开发环境说明文档
