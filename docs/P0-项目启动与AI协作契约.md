## 一、本阶段目标

1. 完成组队，明确团队成员的角色分工
2. 选择并搭建 AI 工具链
3. 签署《AI 协作契约》，建立团队的 AI 使用规范
4. 搭建项目基础设施（Git 仓库、CI 流水线）

---

## 二、任务说明

### 任务 1：组队与角色分工

4 人一组，确定初始角色分工：

| 角色 | 职责 |
|------|------|
| 需求负责人 | 负责需求获取、文档撰写、用例建模 |
| 架构负责人 | 负责技术选型、架构设计、ADR 撰写 |
| 开发负责人 | 负责编码规范、代码审查、核心模块开发 |
| 测试负责人 | 负责测试策略、测试用例设计、质量保障 |

**要求：** 制作一份角色分配计划表，每位团队成员需在文档中简要说明

1. 对所承担角色的理解
2. 该角色的核心职责与目标

### 任务 2：AI 工具选型

团队讨论并选择本项目使用的 AI 工具链。需要考虑：

**需要选择的工具类型：**
1. **AI 编程助手**（如 GitHub Copilot、Cursor、Claude Code、通义灵码等）
2. **AI 对话工具**（如 ChatGPT、Claude、文心一言等，用于文档生成、需求分析）
3. **AI 代码审查工具**（如 CodeRabbit、AI Code Review 等）
4. **AI 测试工具**（如 AI 测试用例生成工具）

**选型要求：**

- 每种类型至少评估 2 个候选工具
- 记录选择理由（功能、易用性、成本、团队熟悉度）或不选择某工具的理由

### 任务 3：签署《AI 协作契约》

团队共同制定并签署一份《AI 协作契约》，明确以下内容：

```
《AI 协作契约》模板

团队名称：__________
日期：__________

第一条：AI 使用原则
- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

第二条：AI 使用范围
- 允许使用 AI 的场景：__________
- 禁止直接使用 AI 输出（需额外审查）的场景：__________
- 完全禁止使用 AI 的场景：__________

第三条：过程记录要求
- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]
- 每阶段填写《AI 协作反思日志》

第四条：代码合入规则
- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数

第五条：违约处理
- 团队内部约定的违约处理方式

全体成员签名：
```

**思考引导：** 为什么需要这份契约？如果没有审查规则，可能会出现什么问题？请回忆课堂上讲到的"工具依赖而能力退化"现象。

### 任务 4：搭建项目基础设施

1. **创建 Git 仓库**
   - 初始化项目仓库（GitHub/GitLab/Gitee 均可）
   - 配置 `.gitignore`（根据选择的技术栈）
   - 编写 `README.md`（项目简介、团队成员、技术栈）
   - 确定分支策略（建议：main + dev + feature 分支）

2. **搭建基础 CI 流水线**
   - 配置自动化构建（可使用 GitHub Actions / GitLab CI）
   - 至少包含：代码编译检查 + 代码风格检查

3. **初始化项目结构**
   - 创建基础目录结构
   - 配置开发环境说明文档

## 三、AI 反思任务

本阶段不要求提交正式的反思日志，但请完成以下思考练习：

> **练习：** 尝试用 AI 生成一份"团队章程"或"项目 README"的初稿。然后对比团队实际讨论后的版本。思考：
> - AI 生成的内容有哪些是通用的套话？
> - 有哪些是只有你们团队自己讨论才能确定的内容？
> - 这个对比对后续使用 AI 有什么启示？
